# 生产受控验收交接清单

**定义(`docs/02` §生产受控验收)**:发布门禁达标后,对生产环境**真实资金通道、充值回调、发货幂等、生产密钥和平台日志检索**的受控确认;**不属于 SDK 继续开发范围**,不能用产生测试业务数据的生产 smoke 替代。

本清单把上述事项逐项移交给对应责任方。SDK/服务端一侧在里程碑 4 已把"结构正确、能力纯净、fail-closed"做到位(见下"SDK 侧已就位");多数项的**真实链路确认**由平台/运维/游戏服务端在受控条件下完成。**但第 2/3/5 项另有服务端开发前置(#57 业务日志、#58 生产回调触发/接收端),须先补齐再做受控确认——这部分属服务端开发,不在"范围外"。**

> 关联:PRD #33(里程碑 4)、M5 规划 #32。本清单为 SDK 范围与生产受控验收的边界交接物。

## SDK 侧已就位(里程碑 4 交付,可作交接前提)

- 生产服务端已上线 CTID 106 / `sdk.xingninghuyu.com`,结构冒烟 8/8(`scripts/smoke-prod.sh`):`/healthz` 我们的服务、`/internal/*` 与 `/pay/*` 生产 404、无签名 `/api/sdk/v2/config` → 401 `signature_invalid`。
- 生产启动 **fail-closed**:缺 `CALLBACK_SECRET` / `SIGNING_KEY_ID` / `SIGNING_KEY_SECRET` 拒绝启动。
- 独立生产库 `m5755_prod`(与 dev `m5755_v2` 物理隔离)。
- 交付 AAR 纯净化门禁五维 + 元测试通过;充值回调**发送端**(投递 + 签名 + 重试 + 幂等去重)在服务端就位且有测试(`docs/04`/`docs/05`)。⚠️ **但其触发器 `CompletePayment` 现仅挂 dev 控制面、生产构建排除,生产无支付成功接收端点**——生产真单支付后无路径触发回调,须先补(#58)。
- ⚠️ **服务端业务日志缺位**:`internal/` 下单/回调链路零业务日志,日志检索(第 5 项)须先补这段开发(#57)。
- 当前生产用**占位密钥**;真实密钥注入后即满足受控验收前提。

## 交接清单

| # | 事项 | 责任方 | 依赖/前提 | 受控验收方式 |
|---|---|---|---|---|
| 1 | **生产密钥检索与注入** | 运维 | 生产 `SIGNING_KEY_ID`/`SIGNING_KEY_SECRET`/`CALLBACK_SECRET` 与生产库凭据 | 替换 `/opt/m5755/.env` 真实密钥并 `rc-service m5755-server restart`;复跑 `smoke-prod.sh` 仍 8/8;签名请求可通过 |
| 2 | **真实资金通道联调** | 平台 + 资金/支付运营 | 第 1 项;真实支付渠道商户号与回调地址;**⚠️ 开发前置:生产支付页/跳转替换 dev `/pay/` 占位(#58)** | 受控小额真单走通下单→支付→平台入账,确认金额/订单一致 |
| 3 | **平台→游戏服务端充值回调真实链路** | 平台 + 游戏服务端 | 第 2 项;游戏服务端回调验签密钥 `CALLBACK_SECRET`;**⚠️ 开发前置:生产支付成功接收端点 + `CompletePayment` 触发器(#58)** | 真单支付后平台向游戏服务端发起充值回调,游戏服务端验签通过并据此发货 |
| 4 | **发货幂等验证** | 游戏服务端 | 第 3 项 | 同一充值回调重复投递,游戏服务端只发货一次(以 CP 订单号/平台订单号去重);**SDK/客户端支付回调不参与发货判定** |
| 5 | **平台日志检索** | 运维 / 平台 | 生产日志通道(`/var/log/m5755-server.log` 及平台侧);**⚠️ 开发前置:服务端业务日志(下单/回调链路当前零业务日志,#57)** | 能按订单号/账户检索到下单、回调、发货链路日志,异常路径可定位 |
| 6 | **实名核验真实厂商对接** | 平台采购 + 平台 | 真实实名核验源接入(生产 `RealNameMock=false` 已 fail-closed) | 接入真实 provider 后,真实身份信息核验通过/拒绝符合预期;未配置时实名提交明确失败 |

## 边界重申(避免越界)

- 以上各项的**真实链路受控确认**不在 SDK 继续开发范围;不得以"产生测试业务数据的生产 smoke"替代真实受控验收(`02` 红线)。**例外:第 2/3/5 项另有服务端开发前置(#57/#58),属服务端继续开发、须先补齐——不要把它们误当作纯运维/联调项。**
- 客户端支付回调仅 UI 提示,**不是发货依据**;发货唯一依据是第 3 项的真实充值回调,幂等由第 4 项保证。
- 生产环境**禁止创建测试账号/订单**;受控验收的真单为最小必要、可对账、可回收。
