# 生产受控验收交接清单

**定义(`docs/02` §生产受控验收)**:发布门禁达标后,对生产环境**真实资金通道、充值回调、发货幂等、生产密钥和平台日志检索**的受控确认;**不属于 SDK 继续开发范围**,不能用产生测试业务数据的生产 smoke 替代。

本清单把上述事项逐项移交给对应责任方。SDK/服务端一侧在里程碑 4 已把"结构正确、能力纯净、fail-closed"做到位(见下"SDK 侧已就位");各项的**真实链路受控确认**由平台/运维/游戏服务端完成,该确认本身不属 SDK 继续开发范围(`02` 定义)。**服务端开发前置 #60(生产入站支付链路:自建收银台 + 微信/支付宝预下单 + webhook 接收端 + 漏发自愈)已闭合并合并 main(PR #65);商户资质亦已通过(2026-06-14)。第 2/3 项确认的剩余开发前置为:微信预下单出网接线(代码,支付宝 wap 已就绪)+ AAR 支付域外跳 #61——属受控确认前的开发工作、不属"受控确认"本身,不与 `02`「受控确认非开发」冲突。(第 5 项业务日志 #57 已闭合。)**

> 关联:PRD #33(里程碑 4)、M5 规划 #32。本清单为 SDK 范围与生产受控验收的边界交接物。

## SDK 侧已就位(里程碑 4 交付,可作交接前提)

- 生产服务端已上线 CTID 106 / `sdk.xingninghuyu.com`,结构冒烟 8/8(`scripts/smoke-prod.sh`):`/healthz` 我们的服务、`/internal/*` 与 `/pay/*` 生产 404、无签名 `/api/sdk/v2/config` → 401 `signature_invalid`。
- 生产启动 **fail-closed**:缺 `CALLBACK_SECRET` / `SIGNING_KEY_ID` / `SIGNING_KEY_SECRET` 拒绝启动。
- 独立生产库 `m5755_prod`(与 dev `m5755_v2` 物理隔离)。
- 交付 AAR 纯净化门禁五维 + 元测试通过;充值回调**发送端**(投递 + 签名 + 重试 + 幂等去重 + **漏发自愈巡检**)在服务端就位且有测试(`docs/04`/`docs/05`)。✅ **触发器已就位(#60 合并 main,PR #65):生产 webhook 接收端 `/pay/wxnotify`·`/pay/alinotify` 验签/反欺诈/幂等 → `CompletePayment` → 充值回调;投递失败由平台侧巡检重投自愈,不依赖渠道重推。**
- ✅ **服务端业务日志已就位(#57 已闭合)**:`CreateOrder`/`CompletePayment`/`dispatchCallback` 结构化 slog + `accessLog` 中间件,可按订单号/脱敏 account 检索;第 5 项的开发前置已消除,余运维侧日志通道接入。
- **商户资质已通过(2026-06-14)**,真实密钥(平台签名 + 微信/支付宝商户)可注入;注入后即满足受控验收前提(当前仍为占位密钥)。

## 边界重申(避免越界)

- 以上各项的**真实链路受控确认**不在 SDK 继续开发范围;不得以"产生测试业务数据的生产 smoke"替代真实受控验收(`02` 红线)。**服务端开发前置已闭合:#57 业务日志、#60 生产入站支付链路(收银台 + 微信/支付宝预下单 + webhook 接收端 + 漏发自愈,合并 main PR #65);商户资质已通过(2026-06-14)。剩开发前置:微信预下单出网接线(代码,支付宝已就绪)+ 客户端 #61(AAR 支付域受限外跳,拉起渠道 App);余密钥注入/部署等运营前置。这些前置是受控确认的前置开发、不属"受控确认"本身(故不与 `02`「受控确认非开发」冲突)。**
- 客户端支付回调仅 UI 提示,**不是发货依据**;发货唯一依据是第 3 项的真实充值回调,幂等由第 4 项保证。
- 生产环境**禁止创建测试账号/订单**;受控验收的真单为最小必要、可对账、可回收。
