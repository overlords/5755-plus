# 领域文档

工程类 skill 探索本仓库时,应按本文规则消费领域文档。

## 探索前先读

- 仓库根目录的 **`CONTEXT.md`**(目前尚未创建);
- **`docs/adr/`** — 阅读与即将处理的领域相关的 ADR(目前尚未创建)。

若这些文件不存在,**静默继续**,不要提示缺失,也不要主动建议创建。生产者 skill(`/grill-with-docs`)会在术语或决策实际敲定时惰性创建它们。

## 本仓库特有规则:术语表优先

在 `CONTEXT.md` 创建之前(以及创建之后与之并行),领域词汇一律以 **`docs/02-terminology.md`** 为准——它是全项目文档与代码命名的唯一口径(114 条术语 + 6 组易混淆对照)。skill 在命名 issue 标题、重构提案、假设或测试时,必须使用该术语表中的定义,不得漂移到术语表明确避免的同义词(例如不写 token 而写"登录令牌",不用 `account` 指代 5755 账户)。

如果需要的概念尚未进入术语表,这是一个信号:要么你在发明项目不使用的语言(请重新考虑),要么存在真实缺口(记录下来,交给 `/grill-with-docs` 处理)。

