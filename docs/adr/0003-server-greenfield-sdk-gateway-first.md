# 服务端在本仓库从零重构,范围从 SDK 网关面起步

dev.xingninghuyu.com 背后的现有实现是 `~/Developer/U10`(5755 开放平台原型:Node/TS,SDK 网关 + React 运营后台 + 文档站 + 包审核/密钥治理/回调重试等)。我们决定:**不迁移 U10**(否决了 subtree 合史与快照拷贝两种迁移方案),而是在本仓库内参照 U10 与本项目需求**从零重构服务端**,与 SDK 同仓、同步演进——服务端网关 + SDK 是产品核心底座。

一期范围只含 SDK 网关面:04 文档的 `/api/sdk/v1/*` 端点、充值回调(服务端→游戏服务端)、`/api/mock/pay` 调试端点、回归所需的异常注入控制端点,以及支撑它们的最小数据模型。运营后台、文档站、包审核等不进当前范围(8 份权威文档对它们没有任何需求定义,做了无从验收);后期扩展须新文档先行。

理由与 ADR-0001 同构:权威文档(尤其 04 契约)是唯一口径,新服务端从出生起以"逐字段实现 04 契约"为定义;迁移原型会把未经文档约束的实现细节变成事实口径。U10 保持原地,作为阅读参考与现行 dev 部署的承载者。
