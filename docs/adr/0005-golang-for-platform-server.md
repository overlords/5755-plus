# 平台服务端采用 Go

新平台服务端(见 ADR-0003)选用 **Go** 实现,而非沿用参考实现 U10 的 Node/TS 同栈方案。这是项目方选型决定;Go 的单二进制部署、低运行时占用与并发模型对网关类服务是常规优势。

接受的代价:放弃与 U10 逐文件比对的便利——参照旧实现需要跨语言阅读。该成本由 04 契约兜底:v2 契约是字段级明确的 JSON 规格,平台服务端以"逐字段实现 04"为定义,U10 只在业务规则存疑时作行为参考,不作代码参考。

推论:U10 的 Prisma/Postgres 持久化形态不直接平移,数据模型按"一期 SDK 网关面最小数据模型"在 Go 生态内重新选型(实现期决定);部署脚本(push-dev-server)思路可借鉴但需为 Go 产物重写。
