# WebView 加载态反转为品牌旋转 spinner,破 §1.11「无循环动效」一处例外

07 §1.11(2026-06-12 决定)立了一条极简动效基线:界面**禁止弹跳、循环、装饰性动效**,只允许轻微淡入淡出;§1.13(2026-06-13)据此把两个远程 WebView(用户中心 §11.2、协议网页层 §2)的加载态定为「静态占位 + 就绪淡入 + 失败重试」,并明文「**不用 spinner 转圈**」。该版已实现(#51,commit `2b10237`)。

2026-06-13 同日复盘:静态占位虽守住了极简,但加载瞬间品牌存在感弱。决定取设计系统「5755 Gold Orbit」动效,**反转加载态口径**——加载中态改为**轨道环 + 光点绕中心匀速旋转的品牌 spinner**(1.8s/圈、线性、无限循环),斜体「5755」与三点静止;就绪/失败/关抽屉即停转。这是 §1.11「无循环动效」基线**唯一保留的例外**,仅限 WebView 加载态;其余界面仍守原基线。

决定:**WebView 加载态用旋转的品牌 spinner**,以分层 PNG + `ObjectAnimator` 旋转实现(轨道层绕 `pivot=(0.5w, 67/160·h)` 转、「5755」层叠在上不动),**不引入动画依赖**。

理由:
- 加载是用户进入用户中心/协议页的**第一眼**,品牌动效的存在感收益落在高频且显眼的位置;静态占位在此处偏弱。
- 例外**边界清晰且可控**:只开「WebView 加载态」一处,§1.11 明文限定,不会滑坡到全局——其余界面的「无弹跳/循环/装饰动效」不变。
- 旋转用 `ObjectAnimator` 转一张拆层 ImageView 实现,**零新依赖**(守 ADR-0006 最小化 AAR),代码量小、minSdk 21 稳;就绪/失败/detach 即 `cancel`,不空转不泄漏。

被否方案:
- **维持静态占位**(原 §1.13):最省、最极简,但放弃了品牌动效收益;评审认为加载首屏值得这一处例外。
- **Lottie 播原 SVG 动画**:还原度最高(原图本就是 CSS 动画),但要给 AAR 加 `com.airbnb.android:lottie` 依赖 + 体积,**直接撞 ADR-0006**,否。
- **`AnimatedVectorDrawableCompat`**(矢量动画):体积最小、可缩放,但要手转金色渐变/虚线轨道/「5755」描边为 path,有偏离设计风险、工作量大;拆层 PNG 旋转已达成观感,不值得。
- **整图旋转**(「5755」跟着转):数字会翻转,观感差,否;故拆为「轨道转 + 字静止」两层。

推论:
- §1.11 动效条改写:循环动效从「禁止」改为「禁止,唯一例外 = WebView 加载 spinner」,注明反转日期与理由指向本 ADR。
- §1.13 改写:加「口径反转记」blockquote;占位描述由「静态单帧」改为「轨道旋转 + 字/点静止」;资产由单张合并图改为**分层两张**(`m5755_web_loading_orbit.png` 旋转层 + `m5755_web_loading_brand.png` 静止层),两处交叉引用「静态占位」→「品牌动效占位」。
- 资产派生不变量延续:两图皆由设计系统 SVG **去动画 + 去白卡 + 去「加载中」文字后按图层拆分**栅格化派生,透明底、源 SVG 不入库;品牌字以位图像素呈现,不与 §1.11「原生层不内嵌字体」冲突。
- 回归:`WebViewLoadingTest` 仍只硬断言失败→重试路径(已通过);旋转/淡入属视觉态,走人工/模拟器走查(已在 Pixel_4_XL 实测轨道旋转、「5755」正立)。
- 将来若要再加界面动效,**须显式修订 §1.11 并评估是否再开例外**,不得以本 ADR 为先例默认放行。
