# 上线阻断回归以模拟器仪器化测试为主形态

08 文档要求自动化的上线阻断回归(含 12 类异常路径),但未规定运行层。我们决定以**模拟器仪器化测试**(`connectedAndroidTest`)作为回归主形态,而不是 JVM 本地 JUnit:回归覆盖 UI 到状态机的真实全链路,避免"状态机断言通过但 UI 没接上"的盲区;代价是回归门禁依赖可用模拟器、单轮分钟级。

被否的替代方案:纯 JVM 回归(状态机 + 模拟网关注入)——更快,但只验状态机层,UI 接线正确性完全依赖 08 的人工场景走查。

推论:状态机与 Android 框架解耦不再是测试基建的硬要求;`FakePlatformGateway` 类模拟物只允许存在于 test source set,不进入生产源码(纯净化检查从结构上保证,而非字符串扫描兜底)。
