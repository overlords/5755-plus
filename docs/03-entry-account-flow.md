# 03 进入与账号流程

本文是 5755 Android SDK 最小化 AAR 首启进入、登录与账号体系的权威流程规格,规定从游戏启动到游戏进入的完整状态顺序、各节点的阻断行为与回退规则、游戏小号体系、账号变化事件收敛和登出语义。SDK 状态机、业务配套 UI、诊断与回归均以本文为准。

## 1. 主流程顺序

最小链路状态顺序固定为:

```
init(配置拉取 → 渠道诊断 → 接入自检输出)
  → 维护门禁判断
  → login(由游戏调用触发)
  → 协议告知(首次安装或协议版本升级时)
  → 5755 账户登录(短信验证码 / 密码)或自动登录
  → 5755 账户有效检查
  → 实名认证检查
  → 防沉迷进入游戏门禁
  → 游戏小号选择 或 默认游戏小号自动进入提示
  → 返回当前游戏小号 ID + 登录令牌
  → 游戏服务端登录态校验
  → 游戏进入
```

`login` 由游戏端调用触发。SDK 不能在 `init` 成功前进入协议告知、5755 账户登录、游戏小号选择或登录成功态。

## 2. 各节点规格

### 2.1 init

`init` 是后续一切业务 API 的明确前置。初始化必须完成配置拉取、渠道诊断、接入自检输出和维护门禁判断;阻断型自检项失败时 `init` 不能成功,诊断型自检项(如渠道异常)只输出诊断并回退默认值,不阻断后续链路。

### 2.2 维护门禁

维护门禁由平台配置驱动,在初始化成功后、协议告知或自动登录之前执行。维护开启时,SDK 展示维护提示并阻断进入流程。**维护阻断不触发账号变化**——游戏不得把维护误判为账号失效。

### 2.3 协议告知

- 触发条件为安装级和协议版本级:新安装、清除数据、卸载重装或协议版本升级后,`login` 必须强制经过协议告知前置状态。
- 同一安装内协议已同意且协议版本未升级时,自动登录、退出登录后重新登录和切换游戏小号都不重复展示协议告知。
- 玩家拒绝协议时,SDK 阻断进入流程;**不杀游戏进程,不触发账号变化**(拒绝发生在账号产生之前,不存在需要清理的游戏会话)。

### 2.4 5755 账户登录与自动登录

- 5755 账户登录支持短信验证码和密码两种方式。SDK UI 不表达"注册成功"语义:验证码登录下是否创建新平台主账户由服务端识别,客户端口径为"5755 账户登录成功后进入实名检查"。
- 已有本地会话时执行自动登录。**自动登录失败时,SDK 必须回退展示 5755 账户登录窗口**,不得以登录失败码直接结束登录流程。这是固定规格:任何"无有效本地会话/自动登录不通过"的路径,终点都是 5755 账户登录窗口,而不是死状态。
- 本地登录态或 token 只能用于发起自动登录,不能替代账户有效检查放行进入游戏;网络失败或平台不可用时不能用本地登录态放行。
- **自动登录有指示,不静默**(#6):自动登录通过 5755 账户有效检查后、进入实名/防沉迷门禁之前,给出轻提示 `正在自动登录…`(取到昵称时为 `正在以 {昵称} 自动登录`)再继续,不得无提示直接跳进门禁。

### 2.5 5755 账户有效检查

5755 账户登录或自动登录成功后,必须执行 5755 账户有效检查。检查失败时返回 5755 账户登录窗口;若已有当前游戏小号状态需要清理,通过账号变化通知收敛。

### 2.6 实名认证

实名认证固定发生在 5755 账户登录成功后。实名状态归属于 **5755 账户**,不归属于游戏小号、角色或游戏服务端账号。未实名的 5755 账户必须先完成实名认证,才能继续后续流程;实名通过后流程继续,玩家不需要重复登录。

### 2.7 防沉迷进入游戏门禁

防沉迷进入游戏门禁在实名认证检查之后执行。实名认证和进入游戏门禁是两个独立节点:实名通过不等于一定可以进入游戏。门禁阻断时展示防沉迷提示并阻断进入流程,**不触发账号变化或登出**,并输出诊断供区分"防沉迷限制"和"登录失败"。

防沉迷支付门禁与进入游戏门禁分离,只在支付前置中执行:阻断时只失败本次支付并提示限制,同样不触发账号变化或登出。可进入游戏不等于一定可支付。

### 2.8 游戏小号选择 / 默认游戏小号自动进入提示

门禁放行后,按是否已显式设置默认游戏小号分支:

- **未设置默认游戏小号**:展示游戏小号选择页,由玩家确认进入(详见第 4 节)。
- **已设置默认游戏小号**:展示轻量的小号自动进入提示。提示显示当前默认游戏小号和切换入口;玩家不操作则继续返回默认游戏小号和登录令牌;点击切换进入游戏小号选择页;取消切换保持当前默认小号。
- 小号自动进入提示只属于 `login` 流程,不属于用户中心,也不是完整游戏小号选择页。
- **上次游戏小号不能作为自动进入依据**。它可以是历史记录,但不是玩家的默认授权。

### 2.9 返回结果与服务端登录态校验

登录成功后,SDK 返回当前游戏小号 ID(`User.getAccount()`)和登录令牌(`User.getToken()`)。游戏服务端使用这两项按协议完成登录态校验后,游戏才进入。SDK 只做登录态校验配合(返回凭据 + 归属敏感动作前的本地已登录且令牌非空前置检查),不在客户端替代服务端登录态校验。

## 3. 阻断与回退规则汇总

| 节点 | 阻断/失效情形 | 行为 | 是否触发账号变化 |
| --- | --- | --- | --- |
| init | 阻断型自检失败、配置失败 | `init` 失败并输出诊断,不进入登录流程 | 否 |
| 维护门禁 | 平台配置维护中 | 展示维护提示,阻断进入流程 | **否** |
| 协议告知 | 玩家拒绝协议 | 阻断进入流程;不杀游戏进程 | **否** |
| 自动登录 | 无有效本地会话 / 5755 登录失败 | **回退展示 5755 账户登录窗口** | 否 |
| 账户有效检查 | 5755 账户失效 | 清理当前游戏小号,返回 5755 账户登录窗口 | 是 |
| 实名认证 | 未实名 | 强制进入实名认证,通过前不得选择游戏小号 | 否 |
| 防沉迷进入游戏门禁 | 合规阻断 | 展示防沉迷提示,阻断进入流程 | **否** |
| 防沉迷支付门禁 | 合规阻断 | 仅失败本次支付并提示限制 | **否** |
| 游戏小号失效 | 当前小号失效 | 进入游戏小号选择页(**不**返回 5755 账户登录窗口) | 是 |
| 5755 账户失效(运行期) | 踢号、令牌失效等 | 清理当前游戏小号,返回 5755 账户登录窗口 | 是 |

5755 账户失效与游戏小号失效是两类独立路径:前者回 5755 账户登录窗口,后者进游戏小号选择页,不得混用。

## 4. 游戏小号体系

### 4.1 小号列表

- 游戏小号选择页必须拉取并展示**当前 5755 账户、当前游戏下的真实游戏小号列表**,不允许使用演示小号列表表达业务流程。
- 新玩家实名通过后,**首个真实游戏小号由平台服务端内部保障并通过小号列表返回**;SDK 不发起单独的首个小号创建接口。列表为空时 SDK 阻断登录并输出诊断,不本地伪造小号。
- 没有默认小号时,即使列表只有一个小号,仍展示小号选择页由玩家确认进入。
- 玩家后续自行创建的游戏小号同样出现在真实列表中。

### 4.2 添加游戏小号

添加游戏小号是当前 5755 账户、当前游戏下的子账号创建动作:

- 添加上限为 **10 个**;达到上限时只提示原因,不允许继续添加。
- 添加成功后刷新小号列表;**不自动进入游戏,不自动设为默认**。

### 4.3 默认游戏小号

- 默认游戏小号**只能由玩家点击小号行上的默认标签显式设置**;点击小号行进入不自动设置默认。
- 默认标签设置后立即展示勾选状态;选中态跟随 5755 黄色品牌规范,不得复用错误、危险或退出状态的视觉表达。
- 只有已设置默认游戏小号时,后续 `login` 才允许展示小号自动进入提示。

### 4.4 选择页 UI 要点

- 页面顶部展示当前 5755 账户昵称,帮助玩家确认主账户身份。
- 页面右侧切换按钮进入 **5755 账户登录页**,重新登录后继续执行账户有效检查、实名认证、防沉迷门禁和小号选择;该入口不是用户中心,也不是用户中心里的切换游戏小号。
- 右上角关闭按钮压在小号选择页上层,圆心对齐选择页右上角,描边使用浅灰样式。关闭语义按链路区分:登录进入链路关闭表示进入未完成;切换小号链路关闭表示取消切换并保持当前游戏小号。

### 4.5 切换游戏小号

- 切换游戏小号(无论来自用户中心还是其他入口)必须先进入游戏小号选择页,由玩家明确选择目标小号后才完成切换;**不展示小号自动进入提示,不允许静默自动切换到下一个小号**。
- 取消切换时保持当前游戏小号,当前游戏会话不受影响。
- 没有当前游戏小号时,不得为了切换而隐式触发初始化、登录或退出登录。

## 5. 账号变化事件收敛

以下事件统一收敛到用户监听(`UserListener`)的账号变化/登出路径,游戏侧用同一入口清理本地会话:

- 玩家退出登录
- 切换游戏小号
- 踢号
- 5755 账户失效
- 游戏小号失效

以下阻断**不属于**账号变化,不得触发该路径:维护门禁阻断、协议拒绝、防沉迷进入游戏门禁阻断、防沉迷支付门禁阻断。

用户中心 H5 内发生的退出登录、切换游戏小号或账号失效类动作,经 JS Bridge 回传 SDK 后,同样收敛到账号变化通知。

## 6. 登出语义

- **退出登录(logout)**:清理 5755 登录态和当前游戏小号,触发账号变化通知,返回 5755 账户登录窗口;**不重新展示协议告知**(同一安装内协议已同意且版本未升级)。
- **用户中心退出登录**:语义同上。用户中心的"切换账号"语义是切换游戏小号,不是退出 5755 账户;只有退出登录才返回 5755 账户登录窗口。用户中心不提供切换 5755 账户——主账户切换只能通过退出登录后重新登录完成。
- **退出游戏(shouldQuitGame)与销毁(destroy)**:作为生命周期交接 API 保留,用于游戏退出确认和 SDK 资源释放,不改变账号状态语义。
