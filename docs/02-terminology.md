# 5755 SDK 术语表

> 本术语表是全项目文档与代码命名的唯一口径。文档、公开 API、内部模块和验收材料中的命名与语义判定均以本文为准；冲突表述一律修正。

## 1. 基础概念

**游戏**：接入 5755 平台的宿主移动游戏，由平台分配的游戏 ID 标识。不称 App、应用。

**接入方**：负责把游戏接入 5755 SDK 的开发者或发行侧团队。不是 SDK 方，也不是玩家。

**游戏 ID**：平台分配给游戏、用于绑定平台配置和运营服务的标识。不写作 appId、game_key。

## 2. 账户与登录

**5755 账户**：玩家在 5755 体系内的主账户；其下可创建多个游戏小号，实名防沉迷状态归属于它。对外不称 account（account 留给游戏小号）。

**5755 账户登录**：SDK 登录流程中完成主账户身份确认的动作；成功后才能继续账户有效检查、实名检查、防沉迷进入门禁和小号选择。游戏只接收当前游戏小号 ID 和登录令牌，不接收 5755 账户 ID。

**5755 账户登录方式**：登录页的身份确认方式，当前为验证码登录和密码登录。它不表示 SDK 暴露单独注册通道；账号是否需平台侧创建由服务端识别，SDK 对外统一按登录流程推进。

**短信验证码请求**：验证码登录前请求平台向输入手机号发送验证码的动作；只用于取得本次登录凭据，不等于登录成功、注册入口或设备验证。

**5755 账户昵称**：登录流程中供玩家确认主账户身份的昵称；只用于 SDK UI 展示，不对游戏开放为 `account`，不等于当前游戏小号 ID。

**自动登录 5755 账户**：老用户再次启动同一已同意协议的游戏时，SDK 基于本地有效会话发起账户恢复。本地登录态只用于发起自动登录，不能替代真实账户有效检查；之后仍须检查账户有效性、实名和防沉迷门禁。

**5755 账户有效检查**：对本地账户会话是否仍可继续登录流程的检查；未通过时返回账户登录窗口，不重复展示已同意的协议告知。网络失败或平台不可用时不能用本地登录态放行，应展示阻断状态。

**登录令牌**：平台登录成功后返回、把当前游戏会话关联到游戏小号的凭证。不写作 token、MTOKEN。

**登录态校验**：游戏服务端使用游戏小号和登录令牌向平台确认登录状态有效的动作；客户端登录成功不等于校验通过。

**登录态校验配合**：SDK 提供当前小号 ID、登录令牌和本地前置检查以协助上述校验；SDK 客户端不能替代服务端确认。

**游戏进入**：游戏服务端完成登录态校验后允许玩家进入游戏内容的业务结果；不等于 SDK 登录成功或小号选择完成，不是客户端本地放行。

## 3. 游戏小号

**游戏小号**：5755 账户下与某游戏关联的子账号，是 SDK 和后端接口对外暴露的 `account` ID；对接入方即该游戏内唯一用户，用于登录态校验、角色上报和支付归属。平台主账户标识必须使用区别于 `account` 的字段名。

**当前游戏小号**：玩家最终选择或确认进入游戏的小号；接入方只接收它，不接收小号列表。本地保存的当前小号不能在有效检查失败、网络失败或平台不可用时放行进入游戏。

**上次游戏小号**：上一次成功进入该游戏时使用的小号；不等于默认游戏小号，不能单独作为自动进入依据。

**默认游戏小号**：被玩家在选择页显式勾选为默认的小号；点击小号进入不会自动设默认。只有已设默认时才可作为自动进入提示的依据，且不能替代真实小号有效检查。

**默认标签**：小号行上的显式设默认入口，点击后立即展示勾选状态；选中态表示确认默认，不表示错误、危险或退出。点击小号行进入游戏不等于点击默认标签。

**首个游戏小号保障**：新用户首次进入当前游戏时，第一个真实小号由平台侧保障并随登录结果和小号列表返回，不需创建引导，也不由 SDK 发起创建接口。首个小号存在不等于默认小号已设置；无默认时仍进选择页由玩家确认。

**添加游戏小号**：玩家在选择页主动创建当前账户、当前游戏下新小号；不等于注册新 5755 账户或创建游戏内角色。最小上线口径：平台生成名称并刷新选择页，新小号不自动进入也不自动设默认，须再点击小号行才算选择；已有 10 个时只提示上限。

**添加游戏小号失败**：达上限或网络/平台失败时停留在选择页并提示（后者提示稍后重试）；账户失效时触发账号变化并返回登录页。失败不触发游戏进入、角色上报或支付，也不改变当前/默认小号。

**切换游戏小号**：同一账户、同一游戏内换用另一小号；必须进入小号选择并由玩家明确选择目标，不回到账户登录窗口，也不能静默自动切换。没有当前小号时不得为切换隐式触发初始化、登录或退出登录；取消切换保持当前小号。

**游戏小号选择页**：展示当前账户、当前游戏下真实小号列表的页面（含平台返回的首个小号与玩家自建小号，不含演示小号）；顶部展示账户昵称，切换按钮返回账户登录页。关闭即未选择：登录链路关闭表示进入未完成，切换链路关闭表示取消并保持当前小号。

**默认小号选择页**：无已设默认小号时展示的小号选择页，玩家需选择并可设默认；不是自动进入提示，也不做上次小号恢复。

**小号自动进入提示**：已设默认小号且有效检查通过时短暂展示当前小号与切换入口的轻量提示；不操作则返回默认小号给游戏，点击切换进入切换流程。切换入口不改变默认小号，取消保持当前小号。不是用户中心或完整选择页，也不是静默进入。

**游戏小号有效检查**：对默认/当前/所选小号能否继续作为当前小号进入游戏的检查；未通过时进入小号选择，而不是重新登录账户。网络失败或平台不可用时不能本地放行。

**区服**：角色所在的游戏服务器、服区或分片。不写作 server。

**角色**：游戏小号在某区服中的游戏内角色或进度记录；只有游戏能区分唯一角色时才构成可上报对象。

**角色资料**：登录态校验通过、游戏进入并拥有角色后提供给平台的角色属性集合，用于运营统计、活动计算和支付归属。SDK 不在小号返回后自动上报；游戏无法区分唯一角色时不调用上报。

## 4. 合规门禁（实名 / 防沉迷 / 维护 / 协议）

**协议告知**：新安装的游戏首次触发 SDK 进入流程时展示并要求确认的平台协议、隐私政策或运营商协议；同意后同一安装内不重复，卸载重装、清除数据或协议版本升级后重新告知。未确认时 SDK 不应完成登录或进入游戏。不是每次登录弹窗，也不是登录页隐私勾选。

**协议拒绝**：玩家明确拒绝协议告知，是强制退出游戏进入流程的合规结果，不是普通登录取消；SDK 不需终止进程，但游戏必须不可继续进入。它发生在账户登录和当前小号产生之前，不触发账号变化。

**实名认证**：在 5755 账户层完成的身份确认流程，是防沉迷判定的输入；归属账户，不归属游戏小号，也不是游戏内实名。

**未实名 5755 账户**：已登录但未完成实名认证的账户；须先完成实名，通过后才可选小号进入游戏。

**防沉迷**：按身份和年龄状态限制或放行游玩、支付的合规门禁。不缩写为 anti、FCM。

**防沉迷进入游戏门禁**：决定当前账户能否进入游戏内容的门禁；实名通过不等于一定放行。阻断时展示提示并阻断进入，不回登录窗口或小号选择，不触发账号变化。

**防沉迷支付门禁**：决定当前账户能否发起支付的门禁；可进游戏不代表放行支付。阻断只失败支付并提示限制，不触发账号变化或登出。

**防沉迷阻断态**：账户因防沉迷规则暂不能进入游戏或支付的状态；进入门禁阻断阻止进入流程，支付门禁阻断只失败支付。不是账号失效。

**维护门禁**：平台配置、SDK 在初始化成功后且协议告知或自动登录前执行的游戏进入状态门禁；SDK 拉取配置、展示提示并阻断进入。指游戏暂不允许进入，不是 SDK 自身维护或版本升级；阻断不触发账号变化。

## 5. 支付与订单

**订单**：玩家为游戏商品发起的支付请求，关联金额、商品、CP 订单号、当前小号、区服和角色等显式字段。不称充值、payment。

**CP 订单号**：游戏服务端在调用支付前生成的唯一订单标识；平台在充值回调中返回，游戏侧校验归属和金额。不写作 mark。

**支付订单入参**：接入方通过 SDK 支付订单对象传入的字段集合；支付 UI 展示和支付请求必须以这些字段为准。

**演示订单兜底**：缺少真实订单字段时用固定金额、商品、区服、角色或 CP 订单号继续支付的行为；不允许进入最小上线链路。

**游戏侧支付触发**：玩家在游戏内触发购买后，由游戏基于已获取的当前小号和登录令牌发起 SDK 支付；SDK 不强制校验游戏是否已完成游戏进入，只校验本地登录态、当前小号、登录令牌、防沉迷支付门禁和订单入参。

**SDK 自有支付容器**：承载支付流程展示和跳转的 SDK 自有页面能力；不等同第三方支付原生 SDK，也不是游戏自建支付页。

**支付归属**：订单在平台侧归属于哪个当前小号、区服和角色的业务关联；不是客户端回调结果。

**客户端支付状态**：SDK 客户端对一次支付进展的描述（已发起/取消/失败/处理中）；只用于界面提示和本地流程处理，不能表示充值回调已到达或物品已发放，也不改变账户或小号状态。

**客户端支付回调**：SDK 客户端通知接入方客户端支付状态的回调；不是充值回调，也不是物品发放依据。

**充值回调**：平台确认第三方支付结果后通知游戏服务端的结果消息；与客户端支付回调严格区分。

**充值回调结果**：游戏服务端对充值回调的处理响应；成功表示已完成合法校验和发放，明确失败表示该笔充值无法成功，其他异常由平台重试。

**充值回调支持**：SDK 对充值回调提供的客户端交接口径、订单归属信息、文档、模拟回调验收和诊断支持；不表示 AAR 内置服务端验签、金额校验、归属校验或幂等发放实现。

**物品发放**：游戏服务端确认充值回调合法后交付虚拟币、商品或权益的动作。不称发货、delivery。

**幂等发放**：对同一笔合法充值回调只执行一次发放；重复通知只确认处理结果，不重复交付。

**服务端接入协议**：游戏服务端与平台交互必须遵守的 HTTP 接口、参数、签名、响应和幂等约定；不是 SDK 客户端 API。

## 6. 用户中心与悬浮球

**悬浮球**：SDK 在游戏内提供平台能力入口的浮层控件；最小上线只要求保留用户中心入口。不称 FAB。

**用户中心**：通过悬浮球打开的 H5 页面，供玩家查看或管理当前小号相关账号服务；依赖 SDK 当前小号上下文，不能独立维护另一套当前账号。

**用户中心 H5 容器**：承载用户中心 H5 的最小页面容器，只提供当前小号上下文和必要桥接；不是通用 H5 容器。

**用户中心 JS Bridge**：用户中心 H5 与 SDK 的最小桥接边界，只允许读取当前小号上下文并回传账号动作；不是通用 JS Bridge。

**用户中心账号动作**：用户中心内触发的退出登录、切换小号或账号失效类操作，必须收敛到 SDK 账号变化通知。用户中心不提供切换 5755 账户；切换小号直接进小号选择且不展示自动进入提示，取消保持当前小号；只有退出登录才返回账户登录窗口。

**平台 App**：可在游戏流程中被拉起、查询或下载的 5755 配套应用。不称盒子。

**客服会话**：游戏内面向玩家的客服沟通场景。不称 IM、chat。

## 7. SDK 基础能力

**SDK 基础能力**：不直接构成游戏业务结果、但支撑最小上线链路可用和可校验的能力集合（运行配置、接入自检、包来源归因、网络与安全传输、日志诊断、维护门禁等）。不是可选运营能力或接入方自建能力。

**接入自检**：初始化或上线回归中判断接入方工程、游戏包和运行环境是否满足当前接入产物要求的校验能力；当前只覆盖最小链路所需条件，不检查完整包能力，不替代人工验收。

**阻断型自检项**：会阻止 `init` 成功或阻断后续最小链路的自检项（缺必要游戏 ID、SDK 配置非法、运行基线不满足、关键 Manifest 权限缺失、签名/包规范严重不满足等）。渠道异常不属于此类。

**诊断型自检项**：必须输出诊断但不阻断最小链路的自检项（渠道缺失/不一致、非关键环境信息不可读等）；需降级或回退默认值，不构成初始化失败。

**SDK 轻量网络层**：AAR 内部支撑最小链路请求的网络能力；不作为接入方可选网络框架暴露。

**SDK 轻量页面容器**：AAR 内部承载登录、协议告知、支付、门禁和用户中心 H5 等最小链路页面的自有页面能力；不是 Fragment 体系或通用 H5 容器。

**SDK 轻量键值存储**：AAR 内部保存登录态、配置缓存、渠道诊断和必要运行状态的轻量本地存储；不是数据库或文件缓存。

**协议级安全能力**：保护登录、请求、支付和回调归属的签名、令牌、HTTPS 和校验能力；不是 native 加密库或第三方安全 SDK，更不是客户端发货依据。

**SDK 诊断输出**：为定位最小链路问题生成的日志、错误码、状态快照或校验结果；属基础能力，不是接入方业务 API、埋点或广告归因。

**可采集诊断输出**：样例、流水线、自检报告或排查流程可稳定读取和归档的诊断输出；临时日志和人工截图不算。

**设备标识能力**：围绕设备、包体和环境标识的采集、传递和诊断能力，用于归因、风控和排障；不代表游戏小号、角色或游戏内唯一用户。

**渠道标识符**：平台或分发链路为游戏包标记的来源标识；不代表游戏、当前小号、角色或支付渠道。1-64 个 ASCII 字符，仅字母、数字、下划线、短横线、点号，归一为小写。

**5755 渠道配置键**：从 Manifest `meta-data` 读取渠道的配置键；正式口径 `m5755_channel`，可识别 `m5755.channel`、`com.m5755.channel` 和通用 `channel`；旧品牌键不进入正式 AAR 承诺。

**默认渠道标识符**：渠道不可读、读取失败、格式非法或与运营配置不一致时的平台保留值，固定为 `default`，表示来源未知但不阻断最小链路。接入方不选择渠道写入方式，也不得用透传参数覆盖渠道。

**5755 账户渠道归因**：主账户首次创建或首次建立平台来源记录时以渠道标识符标记来源；登录请求必须携带已解析渠道及来源，服务端可在新账户归因时写入，但老用户后续登录不得覆盖已有归因。不是小号渠道或支付渠道。

**配置缓存**：本地保存的上次真实配置诊断信息（版本、拉取时间、请求 ID）；只能用于排查和诊断展示，不能在本次拉取失败时替代真实配置放行初始化、门禁或进入游戏。

**平台接口环境**：SDK 联调或生产运行连接的平台环境（开发/预发/生产），决定真实平台域名；不等于业务配置内容本身。

**平台不可用状态**：真实口径下无法取得必要真实配置、账户、小号或支付结果时的明确阻断状态；可让初始化失败或停留在维护/不可用提示，但不能继续跑完整演示成功链路。不新增公开错误码，公开回调复用现有错误码，细分原因进入 message 和内部诊断。

**平台网关边界**：SDK 内部隔离业务状态机与平台真实接口调用的边界；状态机只依赖它表达各类结果，不直接承担 HTTP 拼装、签名或环境选择。不是接入方公开 API。

**SDK 运行模式**：区分本地回归、真实联调和生产运行的内部模式边界，用于证明当前链路用模拟响应还是真实接口；不作为接入方业务 API，也不是模拟兜底开关。

**真实联调模式**：验证 SDK 与真实平台接口贯通的运行口径；真实链路失败时不能自动回退到模拟响应伪装成功。

**平台签名配置**：访问平台真实接口的证书、签名算法、密钥标识和版本等发布配置；各环境可不同，但必须由构建/发布配置固定，不可由接入方传入或运行时切换。诊断只能输出环境、配置版本或 keyId，不能输出密钥或可复用敏感 token。

**透传参数**：不进入可发布 SDK 的通用扩展字段。生产 AAR 不保留初始化透传、订单透传或任何借透传控制环境、运行模式、渠道、配置、fixture、支付附加字段的能力；需参与登录、角色、支付或回调校验的数据必须成为显式字段。

## 8. 生命周期与事件

**账号变化**：因退出登录、踢号、账号失效或切换游戏小号导致当前小号失效或变更的 SDK 事件。切换小号触发账号变化，但不是退出登录，文案不得表达为退出成功；防沉迷支付门禁阻断不属于账号变化。

**退出登录**：玩家主动退出 5755 账户登录态；清理当前小号并返回账户登录窗口。不触发协议告知；重新登录后仍须检查实名和防沉迷门禁。不是切换游戏小号。

**5755 账户失效**：账户会话被判定无效、过期或被踢下线；应清理当前小号、触发账号变化并返回账户登录窗口。不是小号失效或防沉迷阻断。

**游戏小号失效**：当前小号被判定不可继续进入游戏；应触发账号变化并进入小号选择页，不返回账户登录窗口。不是账户失效。

## 9. 交付产物与阶段治理

**AAR 接入**：通过 Gradle 可识别的原样 AAR/JAR 依赖引入 SDK（在线或本地原样文件）；不含手动解包或复制源码。

**最小化 AAR**：面向最小上线链路裁剪的单一公开交付 AAR，只含最小业务链路、基础能力和业务配套 UI 所需内容；可由核心模块和 UI 模块源码侧装配，但接入方只依赖一个 AAR。

**完整 SDK AAR**：包含最小链路、二期和可选运营能力的 AAR；不是当前必接包。

**生产 AAR**：公开交付并用于真实联调或线上发布的 AAR；不得包含样例工程、测试代码、fixture、模拟平台网关或静默回退为模拟成功的逻辑。

**联调 AAR**：连接开发/预发环境做真实联调的 AAR；与生产 AAR 同样必须纯净，差异只能来自接口环境、证书、签名配置、密钥标识或日志级别等构建/发布配置，不能由游戏运行时切换；文件名、版本后缀和验收报告必须可区分。

**公开交付 AAR 纯净化检查**：构建任务 `verifyPublicAarPurity`，真实接口接通前也须独立执行，同时覆盖联调和生产 AAR；阻断 `FakePlatformGateway`、`fixture.*`、透传 API、sample/test/mock/demo 命名、模拟成功文案、非白名单依赖及缺失 `artifactType`、`baseHost` 等产物标识。不通过时不得声明真实联调通过。

**能力白名单**：定义最小化 AAR 内容边界；只有入单能力才允许进入当前产物。不是排除清单或默认全量。

**公开 API 白名单**：对接入方暴露的 API 集合，只含最小链路所需入口；命名与 5755 品牌一致，范围外方法和空实现兼容入口不进入。

**5755 公开 API 命名**：包名 `com.m5755` 承载品牌，公开类名用 `Operate`、`Options`、`User`、`Order`、`RoleInfo`、`Listener` 等中性业务名；正式 AAR 不保留旧品牌别名或双套公开 API。

**最小化 Manifest**：AAR 随包声明的权限、组件、queries 和配置，只含能力白名单所需内容。

**游戏包规范校验**：游戏包为满足平台上线要求须通过的包名、签名、目标版本和 ABI 等校验；区别于 SDK 运行依赖。

**构建环境基线**：接入方工程必须满足的最低构建和依赖解析条件；不是推荐开发环境。

**Android 运行基线**：接入产物承诺兼容的最低 Android 运行版本边界；不是 targetSdk 或编译版本。

**业务最小硬依赖**：最小链路完成业务校验必需的能力依赖，缺失会影响初始化、登录、账号变化、登录态校验配合、角色上报、支付、充值回调、登出、防沉迷、用户中心或维护门禁。

**打包硬依赖**：接入产物正确编译、打包、混淆或运行所需的依赖集合；其业务能力即使不在最小范围内，也可能因引用关系必须随包解析。

**可选业务依赖**：仅服务二期或可选能力的依赖，缺失不应阻断最小链路，但仍可能是打包硬依赖。

**SDK 核心模块**：源码侧承载公开 API、初始化、登录、账号变化、登录态、角色、支付、渠道、网络、安全、存储、诊断和接入自检的内部模块；是产物装配结构，不是接入方单独依赖的 AAR。

**业务配套 UI**：随当前业务能力一起被调用、展示和验收的 UI 表现层；页面是否可见取决于业务能力是否进入最小链路，而非 UI 资产是否存在。由核心模块业务状态驱动，不作为接入方绕过核心 API 直调的入口；UI 物料本身不代表能力上线。

**技术壳通过**：AAR 在产物形态、API 白名单、Manifest、依赖和基础样例上通过校验的阶段状态；不表示业务闭环完成或真实联调完成。

**业务闭环完成**：最小链路在业务前置、门禁、归属、诊断、回调边界和上线阻断回归上全部达标的状态；不等于技术壳通过或仅构建通过。

**第二阶段真实平台联调**：业务闭环完成后对真实配置、账户、小号、支付查询和充值回调链路进行对接、校验和收敛的阶段；不自动引入完整包能力、扩大公开 API 或改变游戏小号语义。

**最小真实配置集**：第二阶段只为最小链路提供的真实配置范围（游戏进入状态、维护门禁、防沉迷门禁、渠道一致性、接口环境策略、排障诊断）；不等于完整运营配置中心。

**模拟器演示验收**：进入生产受控验收前，以玩家视角在模拟器走通样例游戏中的真实 SDK 流程，确认可见交互与真实联调结论一致；必须连接真实开发环境，不得用模拟平台、演示小号或本地成功兜底；不替代生产真实资金和充值回调验收。

**生产受控验收**：发布门禁达标后对生产环境真实资金通道、充值回调、发货幂等、生产密钥和平台日志检索的受控确认；不属于 SDK 继续开发范围，不能用产生测试业务数据的生产 smoke 替代。

**上线阻断回归**：发布前必须通过的自动化或样例回归；失败时不能声明业务闭环完成或上线。不是可选测试或人工冒烟。

**异常路径回归**：验证失败、阻断、重复通知或账号变化等非正常链路可诊断、可收敛且不产生错误业务结果的回归；区别于正常链路冒烟。

## 10. 易混淆对照

### 10.1 5755 账户 vs 游戏小号 vs 游戏内角色

| 术语 | 层级 | 边界要点 |
| --- | --- | --- |
| 5755 账户 | 平台主账户 | 实名防沉迷归属于它；游戏拿不到它的 ID；标识字段必须区别于 `account` |
| 游戏小号 | 账户下、某游戏内子账号 | 对外即 `account` ID，是游戏内唯一用户，用于登录态校验、角色上报、支付归属 |
| 游戏内角色 | 小号在某区服的进度记录 | 由游戏维护；能区分唯一角色才可上报，SDK 不自动上报 |

### 10.2 退出登录 vs 退出游戏 vs 账号变化

| 术语 | 边界要点 |
| --- | --- |
| 退出登录 | 主动退出账户登录态：清理当前小号、返回账户登录窗口；不触发协议告知，重新登录仍查实名与防沉迷 |
| 退出游戏 | 游戏进程/内容层面退出，由游戏侧处理；协议拒绝强制退出的是"游戏进入流程"，SDK 不需终止进程 |
| 账号变化 | SDK 事件：退出登录、踢号、账号失效、切换小号都触发。切换小号触发账号变化但不是退出登录；支付门禁阻断不属于账号变化 |

### 10.3 客户端支付回调 vs 充值回调

| 术语 | 方向 | 边界要点 |
| --- | --- | --- |
| 客户端支付回调 | SDK 客户端 → 游戏客户端 | 只通知客户端支付状态，不表示到账，不是发放依据 |
| 充值回调 | 平台 → 游戏服务端 | 确认第三方支付结果后的服务端消息，是验签、归属校验和幂等发放的唯一依据 |

### 10.4 实名认证 vs 防沉迷进入门禁 vs 防沉迷支付门禁

| 术语 | 边界要点 |
| --- | --- |
| 实名认证 | 账户层身份确认，是防沉迷的输入；不归属小号；实名通过不等于门禁放行 |
| 防沉迷进入游戏门禁 | 决定能否进入游戏；阻断只展示提示并阻断进入，不回登录窗口/小号选择，不触发账号变化 |
| 防沉迷支付门禁 | 决定能否发起支付；可进游戏不代表可支付；阻断只失败支付，不触发账号变化或登出 |

### 10.5 协议告知 vs 登录页协议勾选

| 术语 | 边界要点 |
| --- | --- |
| 协议告知 | 安装维度强制合规确认：首次进入展示，同一安装内不重复，卸载重装/清数据/协议升级后重新告知；未确认不得登录或进入游戏；拒绝即强制退出进入流程 |
| 登录页协议勾选 | 登录页内的隐私勾选交互，不承载协议告知的合规口径：不能替代协议告知，也不按安装维度记忆同意状态 |

### 10.6 首个游戏小号保障 vs 默认游戏小号

| 术语 | 边界要点 |
| --- | --- |
| 首个游戏小号保障 | 平台保障新用户在当前游戏下存在首个真实小号并随登录结果返回；SDK 不发起创建；首个存在不等于默认已设置 |
| 默认游戏小号 | 玩家显式勾选默认标签的结果；点击进入不自动设默认；有默认且有效检查通过才走自动进入提示，且不能替代有效检查 |
