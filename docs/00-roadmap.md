> 先读这张地图,再读 `01–08` 规格。本文是里程碑级的全局站位与终点线,**不复述切片**(切片在 GitHub Issues),也**不是第二份 spec**(口径仍以 `01–08` 为准)。状态会变的细节(切片、HITL 待办值)留在 issue 与执行记忆,本文只给"走到哪、往哪去"。

## 分期词汇(唯一口径)

本项目统一用 **里程碑 M1–M5 + GA 终点线** 描述进度。历史文档里的「一期 / 二期」已退役,对照如下:

| 历史词 | 现含义 |
| --- | --- |
| 一期 | v2 SDK 本体(M1–M4),即"最小化 AAR 的可商用功能集" |
| 二期 | 拆分为三类:**M4 已交付**(生产 AAR 纯净化/运行模式隔离)、**GA 受控验收**(真实平台/账户/支付/充值回调线上联调)、**v2 范围外·未来**(真实用户中心 H5、设备标识、广告归因、第三方监控) |

## 三轨总览

```
主轴 · v2 SDK(Android AAR + 平台服务端 SDK 网关面)
  M1 ──► M2 ──► M3 ──► M4 ──►  ◇ GA 受控验收(进行中)
  ✓      ✓      ✓      ✓        客户端 + 服务端就位、GA 前质量加固已织网(横屏/UX 回归/CI);余生产密钥注入/部署/小额真单 + 实名厂商凭据(运营)

支轨 A · 用户中心平台面(uc SPA + 平台对玩家 API 面 /api/uc/v2)
  ✓ dev 贯通 ──► /api/uc/v2 六端点上 dev(CTID 105)+ uc SPA USE_MOCK=false 上线
                (CTID 107),网关铸 token→uc 真读写全链 live 验证;口径 06a / ADR-0010

支轨 B · 公会运营平台(原 M5,独立产品面)
  规划中(#32)──► 复用 M4 归因地基,独立排期,【不阻塞】主轴 GA
```

三条终点线相互独立:**(A)v2 SDK GA** = 生产受控验收 6 项闭合;**(B)用户中心平台面 v1** = uc SPA 真数据贯通(`/api/uc/v2` 全面 + SPA `USE_MOCK=false`)— **dev 已达成(2026-06-13)**,生产 uc 面随主轴 GA 真平台一并落地;**(C)公会平台 v1** = 独立里程碑。互不阻塞 SDK 商用。

> **支轨 A 与 v2 范围**:用户中心平台面是「平台对玩家」侧产物(`/api/uc/v2` 与 SDK 网关面 `/api/sdk/v2` 并列且独立,ADR-0010),**不进 AAR**。SDK 侧只保留「H5 容器 + bridge」(M3 已交付);uc SPA 由 SDK WebView 加载。若将来要把该面纳入 **v2 正式出货**(而非作为独立平台面),须按 `01 §5` 重走能力白名单/公开 API/Manifest/依赖/接入自检评审。

### GA 终点线:生产受控验收 6 项

口径见 `docs/production-controlled-acceptance-handoff.md`(02「生产受控验收」定义)。客户端 AAR 已"结构正确、能力纯净、fail-closed";服务端密钥注入 fail-closed、实名厂商接线、充值回调**发送端 + 漏发自愈巡检**、业务日志(#57 已闭合)、**生产入站支付链路(#60 已闭合并合并 main,PR #65:自建收银台 + 微信/支付宝预下单 + webhook 接收端)**均已就位;**商户资质亦已通过(2026-06-14)**。剩余前置见下:

1. 生产密钥检索与注入(运维)— 资质已通过、真实密钥可注入;fail-closed,注入即可、无需改码
2. 真实资金通道联调(平台+资金运营)— ✅ #60 服务端已闭合、资质已通过;✅ **微信预下单已合并 main**(PR #67)、✅ **AAR 支付域外跳已合并 main**(PR #68 / ADR-0014:仅收银台白名单 scheme、try/catch 零 queries)、✅ **支付宝沙箱端到端实战通过(2026-06-15)**(下单→收银台→wap→沙箱付款→异步通知验签入账→充值回调验签);剩真实密钥注入触发生产真单
3. 平台→游戏服务端充值回调真实链路(平台+游戏服务端)— ✅ #60 已闭合(渠道 webhook → `CompletePayment` → 充值回调 + 漏发自愈巡检);✅ **支付宝沙箱实战已验证充值回调全链**(2026-06-15:平台→mock 游戏服务端验签通过、`callback_redeliver_sweep confirmed=1`,漏发自愈实测;**签名当时为 MD5,已由 ADR-0016 改 HMAC-SHA256**);剩生产真单触发
4. 发货幂等验证(游戏服务端)— 游戏服务端职责;平台侧符合规格
5. 平台日志检索(运维/平台)— 服务端业务日志已就位(#57 已闭合:下单/回调链路结构化 slog,可按订单号/脱敏 account 检索),余运维侧日志通道检索
6. 实名核验真实厂商对接(平台采购)— NPPA 客户端已接线,填 per-game 凭据即可、无需改码

服务端开发前置 **#60**(自建收银台 + 微信/支付宝预下单 + webhook 接收端 + 漏发自愈)**已闭合并合并 main(PR #65 / ADR-0013:渠道异步 notify=充值回调、同步 return=客户端回调、支付宝公钥模式)**;**商户资质已通过(2026-06-14)**。微信预下单出网接线**已合并 main(PR #67)**、**#61** AAR 支付域外跳例外/`01 §5` 评审 + ADR-0014 **已合并 main(PR #68)**、支付宝**沙箱端到端实战通过(2026-06-15)**;**GA 开发前置全部闭合**,剩密钥注入/部署/小额真单受控确认(运营),即达 **v2 SDK GA**。游戏服务端对接面**鉴权统一已全链落地(ADR-0016,#83–#87 全合并 main)**:充值回调签名 MD5→HMAC-SHA256(取代 #73)+ 游戏服务端 `serverKey`(登录态校验与充值回调统一签名、serverKey 仅可调登录态校验)+ 服务端面 OpenAPI/接入文档;**#59** 已缩到真实游戏方对接(serverKey 发放 / 回调验签联调,并行、不阻塞 GA)。

> 经 grill(2026-06-14)细化:支付集成定为「平台自建收银台 + 微信/支付宝直连」,故 #61 须为支付 WebView 开「支付域受限外跳」(拉起微信/支付宝 App)——这会再动一次「最小化 AAR」身份,走 `01 §5` 评审 + ADR。**已落地(PR #68 / ADR-0014):仅收银台白名单 scheme + try/catch 零 queries,侵蚀压到 `01 §4.2` 一处。**

## v2 范围外 · 未来

以下不在 v2 产物内,进入前须重走能力白名单/公开 API/Manifest/依赖/接入自检评审(`01 §5`):设备标识采集或透传(OAID/Android ID)、广告归因、第三方监控;**平台余额/余额支付(平台币体系,玩家预存真实货币、下单从余额扣减)**——经 grill(2026-06-14)定为范围外·可拓展项,v2 不自持余额、走持牌机构直连代收,合规定性(储值=备付金/二清/支付牌照)与前向兼容缝见 ADR-0012;以及 `01 §4.2` 的永久排除项(客服/IM、平台 App 跳转、活动优惠券、强更、加速连点器、通用 H5 文件/媒体/外跳)。(用户中心远程 H5 平台面已立为**支轨 A**,见上;它是平台对玩家侧产物、不进 AAR,故不在「v2 AAR 产物」内但已正式在册。)
