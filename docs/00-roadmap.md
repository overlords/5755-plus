> 先读这张地图,再读 `01–08` 规格。本文是里程碑级的全局站位与终点线,**不复述切片**(切片在 GitHub Issues),也**不是第二份 spec**(口径仍以 `01–08` 为准)。状态会变的细节(切片、HITL 待办值)留在 issue 与执行记忆,本文只给"走到哪、往哪去"。

## 分期词汇(唯一口径)

本项目统一用 **里程碑 M1–M5 + GA 终点线** 描述进度。历史文档里的「一期 / 二期」已退役,对照如下:

| 历史词 | 现含义 |
| --- | --- |
| 一期 | v2 SDK 本体(M1–M4),即"最小化 AAR 的可商用功能集" |
| 二期 | 拆分为三类:**M4 已交付**(生产 AAR 纯净化/运行模式隔离)、**GA 受控验收**(真实平台/账户/支付/充值回调线上联调)、**v2 范围外·未来**(真实用户中心 H5、设备标识、广告归因、第三方监控) |

## 双轨总览

```
主轴 · v2 SDK(Android AAR + 平台服务端 SDK 网关面)
  M1 ──► M2 ──► M3 ──► M4 ──►  ◇ GA 受控验收(进行中·非开发)
  ✓      ✓      ✓      ✓        生产化已就位,等真实资金/密钥/实名厂商闭合

支轨 · 公会运营平台(原 M5,独立产品面)
  规划中(#32)──► 复用 M4 归因地基,独立排期,【不阻塞】主轴 GA
```

两条终点线相互独立:**(A)v2 SDK GA** = 生产受控验收 6 项闭合;**(B)公会平台 v1** = 独立里程碑,有没有它都不影响 SDK 商用。

### GA 终点线:生产受控验收 6 项

口径见 `docs/production-controlled-acceptance-handoff.md`(02「生产受控验收」定义)。SDK/服务端侧已做到"结构正确、能力纯净、fail-closed";剩下是真实链路确认:

1. 生产密钥检索与注入(运维)— 当前为占位密钥
2. 真实资金通道联调(平台+资金运营)
3. 平台→游戏服务端充值回调真实链路(平台+游戏服务端)
4. 发货幂等验证(游戏服务端)
5. 平台日志检索(运维/平台)
6. 实名核验真实厂商对接(平台采购)

闭合即达 **v2 SDK GA**。

## v2 范围外 · 未来

以下不在 v2 产物内,进入前须重走能力白名单/公开 API/Manifest/依赖/接入自检评审(`01 §5`):真实用户中心 H5 平台侧联调、设备标识采集或透传(OAID/Android ID)、广告归因、第三方监控;以及 `01 §4.2` 的永久排除项(客服/IM、平台 App 跳转、活动优惠券、强更、加速连点器、通用 H5 文件/媒体/外跳)。
