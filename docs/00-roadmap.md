# 00 路线图(项目蓝图)

> 先读这张地图,再读 `01–08` 规格。本文是里程碑级的全局站位与终点线,**不复述切片**(切片在 GitHub Issues),也**不是第二份 spec**(口径仍以 `01–08` 为准)。状态会变的细节(切片、HITL 待办值)留在 issue 与执行记忆,本文只给"走到哪、往哪去"。

## 分期词汇(唯一口径)

本项目统一用 **里程碑 M1–M5 + GA 终点线** 描述进度。历史文档里的「一期 / 二期」已退役,对照如下:

| 历史词 | 现含义 |
| --- | --- |
| 一期 | v2 SDK 本体(M1–M4),即"最小化 AAR 的可商用功能集" |
| 二期 | 拆分为三类:**M4 已交付**(生产 AAR 纯净化/运行模式隔离)、**GA 受控验收**(真实平台/账户/支付/充值回调线上联调)、**v2 范围外·未来**(真实用户中心 H5、设备标识、广告归因、第三方监控) |

## 三轨总览

```
主轴 · v2 SDK(Android AAR + 平台服务端 SDK 网关面)
  M1 ──► M2 ──► M3 ──► M4 ──►  ◇ GA 受控验收(进行中·非开发)
  ✓      ✓      ✓      ✓        生产化已就位,等真实资金/密钥/实名厂商闭合

支轨 A · 用户中心平台面(uc SPA + 平台对玩家 API 面 /api/uc/v2)
  ✓ dev 贯通 ──► /api/uc/v2 六端点上 dev(CTID 105)+ uc SPA USE_MOCK=false 上线
                (CTID 107),网关铸 token→uc 真读写全链 live 验证;口径 06a / ADR-0010

支轨 B · 公会运营平台(原 M5,独立产品面)
  规划中(#32)──► 复用 M4 归因地基,独立排期,【不阻塞】主轴 GA
```

三条终点线相互独立:**(A)v2 SDK GA** = 生产受控验收 6 项闭合;**(B)用户中心平台面 v1** = uc SPA 真数据贯通(`/api/uc/v2` 全面 + SPA `USE_MOCK=false`)— **dev 已达成(2026-06-13)**,生产 uc 面随主轴 GA 真平台一并落地;**(C)公会平台 v1** = 独立里程碑。互不阻塞 SDK 商用。

> **支轨 A 与 v2 范围**:用户中心平台面是「平台对玩家」侧产物(`/api/uc/v2` 与 SDK 网关面 `/api/sdk/v2` 并列且独立,ADR-0010),**不进 AAR**。SDK 侧只保留「H5 容器 + bridge」(M3 已交付);uc SPA 由 SDK WebView 加载。若将来要把该面纳入 **v2 正式出货**(而非作为独立平台面),须按 `01 §5` 重走能力白名单/公开 API/Manifest/依赖/接入自检评审。

### GA 终点线:生产受控验收 6 项

口径见 `docs/production-controlled-acceptance-handoff.md`(02「生产受控验收」定义)。客户端 AAR 已"结构正确、能力纯净、fail-closed";服务端密钥注入 fail-closed、实名厂商接线、充值回调**发送端**也已就位。**但第 2/3/5 项含服务端开发前置(非纯运维/联调),GA 闭合前须先补**:

1. 生产密钥检索与注入(运维)— 占位密钥;fail-closed,注入即可、无需改码
2. 真实资金通道联调(平台+资金运营)— ⚠️ 含开发前置:生产支付页/跳转替换 dev `/pay/` 占位(#58)
3. 平台→游戏服务端充值回调真实链路(平台+游戏服务端)— ⚠️ 含开发前置:生产支付成功接收端点 + `CompletePayment` 触发器(发送端已就位、触发器现仅在 dev 面),见 #58
4. 发货幂等验证(游戏服务端)— 游戏服务端职责;平台侧符合规格
5. 平台日志检索(运维/平台)— ⚠️ 含开发前置:服务端业务日志(下单/回调链路当前零业务日志),见 #57
6. 实名核验真实厂商对接(平台采购)— NPPA 客户端已接线,填 per-game 凭据即可、无需改码

服务端开发前置(#57 日志、#58 回调触发/接收端)闭合 + 上述真实链路确认闭合,即达 **v2 SDK GA**。
